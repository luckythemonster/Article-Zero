# 🔒 Security Vulnerability Fix Task: Anthropic API Key Leakage

## 🎯 What
Removed the direct usage of the Anthropic SDK in the browser. Previously, the `DialogueRouter` was initializing the `Anthropic` client with `dangerouslyAllowBrowser: true`.

## ⚠️ Risk
Using `dangerouslyAllowBrowser: true` bypasses CORS protections and exposes the user's API key to the frontend environment, making it vulnerable to interception and misuse.

## 🛡️ Solution
Introduced a local Express proxy server (`server/index.js`) to handle Anthropic API requests. The frontend now sends the user-provided API key via an `Authorization: Bearer` header to the proxy, which then forwards the request securely to Anthropic. Updated `vite.config.ts` and `package.json` to route `/api` calls and run both the Vite frontend and Express server concurrently.
